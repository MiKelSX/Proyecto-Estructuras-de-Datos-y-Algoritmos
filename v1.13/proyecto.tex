\section{Proyecto}

Una de las labores más importantes que se realizan en el computador es
buscar en bases de datos de cualquier naturaleza. Para agilizar este proceso
se sigue una metodología bastante clara. Primero se debe indexar el conjunto
de trabajo para luego resolver las consultas sobre el índice.

En este caso Ustedes van a trabajar en búsqueda indexada en texto, que es
un problema bastante parecido al que resuelve \verb|www.google.com| pero a
una escala diminuta.

El problema consiste en que dado un archivo de texto plano, en adelante
\emph{texto}, usted busque un determinado string, que se denomina \emph{patrón},
en el texto, reportando la línea en que aparece el patrón.
Si un patrón aparece varias veces, entonces debe reportar la lista de todas
las ocurrencias.

Para resolver este problema Ustedes deben seguir la siguiente metodología:
\begin{itemize}
  \item Implemenar la solución del problema utilizando fuerza bruta. Vale
    decir, haciendo un recorrido secuencial línea a línea buscando el patrón
    durante el texto.
  \item Verificar manualmente que los resultados de su versión de fuerza
    bruta son correctos.
  \item Implementar un cliente interactivo que permita cargar el archivo
    de pruebas y hacer búsquedas por fuerza bruta.
  \item Implementar la solución del problema utilizando un índice basado
    en tablas de hash. Si usted se limita a utilizar los diccionarios
    preimplementados del lenguaje tiene 1.5 de descuento.
    Para acceder al puntaje completo, usted debe codificar el índice tabla de hash.
  \item Una vez que tenga el texto indexado, extender el cliente
    para permitir realizar consultas de patrones sueltos sobre el índice.
  \item Verificar que los resultados de su versión indexada son correctos
    contrastándolos con los resultados de la versión de fuerza bruta.
  \item Extender el cliente para permitir la realización de varias consultas
    recibidas en un archivo de consultas, con un patrón distinto por cada línea.
  \item BONO: Si implementa las tres alternativas (fuerza bruta, usar los
    diccionarios del lenguaje, implementar el diccionario) y realiza la
    comparación experimental de las tres alternativas tendrá 1.5 para la
    solemne 1 ó 2.
\end{itemize}
    
\section{Pruebas}

Una vez que tenga implementado el proyecto y verificado que trabaja de
la forma esperada, realice las siguientes pruebas:

\begin{enumerate}
  \item Utilizando el texto de este proyecto, mida el tiempo de la ejecución
    promedio de la búsqueda con fuerza bruta y la o las alternativas implementadas.
  \item Repita el experimento anterior, esta vez utilizando un otro texto que
    ustedes mismos van a buscar/generar.
\end{enumerate}

Considere que los tiempos de ejecución pueden ser extremadamente breves,
por lo tanto, considere realizar muchas repeticiones de la búsqueda, hasta poder
medir una ejecución de unos tres minutos de la fase de búsqueda.
En su informe indique cuántas repeticiones son necesarias para alcanzar
los tres minutos de ejecución y compare la eficiencia de los métodos
implementados.
